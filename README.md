# ミニCRM (UI/UX Case Study)

案件（スポンサー）から請求、入金までの流れを“漏れなく”一元管理するための、小さなチーム向けの顧客関係管理（CRM）アプリケーションです。
このプロジェクトは、B2B業務アプリケーションにおける設計思想と実装能力を示すためのUI/UXケーススタディとして制作されています。

## このプロジェクトについて

### 背景：スプレッドシート運用の限界

小規模チームでのプロジェクト管理では、情報がスプレッドシート、Slack、カレンダー、Google Driveといった複数のツールに散逸しがちです。これにより、以下のような課題が日常的に発生します。

*   **情報の属人化**: 案件の最新状況が担当者の頭の中にしかなく、チーム全体で進捗を共有できない。
*   **情報の分散**: 関連情報を探すために複数のツールを横断する必要があり、状況把握に時間がかかる。
*   **請求プロセスのリスク**: 月末の請求作業が手作業に依存し、請求漏れや入金確認漏れといった事業上のリスクを常に抱えている。

このミニCRMは、こうした「人の注意力」への依存から脱却し、業務が円滑に回るための“運用のOS”となることを目指します。

### ターゲットユーザー

本プロジェクトでは、以下のような人物像をターゲットユーザーとして想定しています。

*   **職種**: 2〜5名規模のデザインスタジオや制作会社における、プロジェクトマネージャー兼任の経営者。
*   **課題**: 複数の案件を同時に管理しながら、クライアントワークと事務作業の両立に追われている。管理業務を効率化し、より創造的な仕事に時間を使いたいと考えている。
*   **ITスキル**: Google WorkspaceやSlackは使いこなせるが、複雑なツールの導入やカスタマイズは避けたい。

## 設計思想：迷わない・漏れない・復旧できる

このCRMは、単なる「情報の台帳」ではありません。日々の運用コストを下げ、業務品質を高めるための「ルールが埋め込まれた画面」として設計されています。すべての機能は、以下の3つの判断基準に基づいて実装されます。

1.  **迷わない**: 画面を開いた瞬間に「次にすべきこと・担当者・期限」が直感的にわかること。
2.  **漏れない**: 条件付きの必須項目やステータス管理により、入力ミスや確認漏れといったヒューマンエラーを未然に防ぐこと。
3.  **復旧できる**: 操作履歴や例外処理の導線を確保し、問題が発生しても「何が起きたか→次にどうするか」を冷静に判断・対応できること。

## 主な機能と画面構成

MVP（Minimum Viable Product）では、中核となる以下の機能と画面を実装します。

### 主要画面

1.  **ホーム画面（ダッシュボード）**: ログイン直後に「今日やるべきこと」と「事業の全体像」を把握できる画面。要対応タスクや重要指標（KPI）を表示します。
2.  **案件一覧画面**: すべての案件を俯瞰し、ステータスや担当者で動的に絞り込み検索ができるテーブル画面。
3.  **案件詳細画面**: 特定の案件に関する情報（基本情報、ステータス、コミュニケーション履歴など）を集約する「唯一の正しい場所」。
4.  **案件登録・編集フォーム**: ストレスなく直感的に案件情報を登録・更新できるフォーム。

### コア・ワークフロー

案件の発生から完了まで、以下のステータスで進捗を管理します。

`見込み` → `提案中` → `契約` → `請求準備` → `請求済み` → `入金済み`

## プロジェクトのスコープ

### MVPで実装する範囲 (In Scope)

*   取引先（Company）、案件（Deal）、請求（Invoice）、活動ログ（Activity）の基本的なCRUD機能
*   上記ワークフローに沿った案件ステータスの遷移管理
*   ダッシュボードの主要KPI（今月の見込み売上、請求予定など）の表示
*   案件一覧画面での検索・フィルタ機能（請求予定月、ステータス、担当者）
*   UI上での簡易的なリマインドアラート

### 今回はやらないこと (Out of Scope)

*   ユーザーごとの詳細な権限管理や監査ログ機能
*   会計ソフトや外部CRMとのAPI連携
*   PDF形式の請求書自動生成や電子契約サービスとの連携

## 技術スタック

*   **フロントエンド**: Next.js (React), TypeScript, Tailwind CSS
*   **UIコンポーネント**: 独自デザインシステム「Yohaku」（[shadcn/ui](https://ui.shadcn.com/) ベース）
*   **バックエンド（想定）**: Supabase (BaaS) または Node.js (Express)
*   **データベース（想定）**: PostgreSQL
*   **デプロイ（予定）**: Vercel

## プロジェクトの構成

このリポジトリでは、ミニCRMアプリケーションと、そのUIを構築するための独自コンポーネントライブラリ「Yohaku」を管理します。

*   `src/components/yohaku`: アプリケーション全体で共通利用されるUIコンポーネント群です。これらは `shadcn/ui` を基盤に、本プロジェクトのデザイン原則に合わせてカスタマイズされています。
*   `src/app` (今後作成): CRMアプリケーション本体の画面やロジックを配置するディレクトリです。

現在の `src` ディレクトリは、主にこの「Yohaku」ライブラリのソースコードによって構成されています。

## 開発環境のセットアップ

**注意**: このリポジトリには `package.json` がまだ含まれていません。開発を始めるには、まず `package.json` を作成し、依存関係をインストールする必要があります。

1.  **`package.json` の作成**
    プロジェクトのルートに `package.json` ファイルを作成してください。（内容は `GEMINI.md` を参考にしてください）

2.  **依存関係のインストール**
    ```bash
    npm install
    # or
    # yarn install
    ```

3.  **開発サーバーの起動**
    ```bash
    npm run dev
    ```
    ブラウザで `http://localhost:3000` を開いてください。
